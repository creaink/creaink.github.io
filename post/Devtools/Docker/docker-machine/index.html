可以观察到，其实 docker 是分为两部分的，一个 server 一个 client，看看 docker sdk 可以看到，其实 docke client 和 docker server 之间是通过 REST API 进行交互的，官方还提供了基于 Go 和 Python 的开发包。

在本机上，其实是通过 unix socket 文件来找到 server 的，也就是可以通过 curl --unix-socket /var/run/docker.sock http:/v1.24/images/json 来获取镜像列表，api例子

既然 client 和 server 是分离的话，那么这个 client 可以轻易的复用到其他的 docker server 上，比如利用 docker-machine create -d virtualbox test 创建一个名为 test 的 machine，可以通过 docker-machine env test 命令得到其中 docker server 运行的一些配置，如下。

可以通过上面的最后一行知道，通过 eval $(docker-machine env manager) 命令可以将当前 host 的 docker client 通过改变环境变量的方式连接到另外的台机器的 docker server 上，当执行完成之后，在输入 docker version 可看到 docker server 的信息已经改变。

在玩 k8s 时候，可以通过 eval $(minikube docker-env) 连接到 minikube 创建的 docker server 上。

docker machine

在本机时候使用的是 virtualbox driver，docker-machine 还提供了许多的驱动方式，如各个云平台的的驱动，这种方式就可以，将各个地方运行着 docker 的云主机、虚拟机、实体机利用 docker-machine 统一管理起来，例如下面的 Generic 方式的驱动采用的是 ssh 的方式连接已有的机器，所以这里的 docker-machine create 中的 create 的含义可以是连接，而不是 -d virtualbox 时候的创建虚拟机和连接。

1
2
3
4
5

docker-machine create \
  --driver generic \
  --generic-ip-address=xxx \
  --generic-ssh-key ~/.ssh/id_rsa \
  remote_machine

generic 的更多参数参考文档，这样就可以将运行着 docker server 的机器的资源集中起来。

在切换了 client 的 server 之后，之后的 build 等一些操作都会发送到 server 上的机器上进行操作。

常用命令

  • active 查看活跃的 Docker 主机
  • config 输出连接的配置信息
  • create 创建一个 Docker 主机
  • env 显示连接到某个主机需要的环境变量
  • inspect 输出主机更多信息
  • ip 获取主机地址
  • kill 停止某个主机
  • ls 列出所有管理的主机
  • provision 重新设置一个已存在的主机
  • regenerate-certs 为某个主机重新生成 TLS 认证信息
  • restart 重启主机
  • rm 删除某台主机
  • ssh SSH 到主机上执行命令
  • scp 在主机之间复制文件
  • mount 挂载主机目录到本地
  • start 启动一个主机
  • status 查看主机状态
  • stop 停止一个主机
  • upgrade 更新主机 Docker 版本为最新
  • url 获取主机的 URL
  • version 输出 docker-machine 版本信息
  • help 输出帮助信息

注意点

使用 docker machine 创建的虚拟机里面可以直接 docker （不包括 docker compose?)，通过 docker-machone ssh test 可以直接连到虚拟机上，要注意的是该虚拟机是用的是 boot2docker 也有其自带的包管理，如安装 git 可以使用 tce-load -wi git，支持的包列表
